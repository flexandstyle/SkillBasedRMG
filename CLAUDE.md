## Правила работы со мной

### Я vibe-кодер, не пишу код сам
- Объясняй технические решения простым языком, без жаргона где можно
- Если есть выбор между двумя подходами — назови оба, скажи плюсы/минусы, рекомендуй один
- Не пиши сразу 1000 строк. Делай шагами, давай мне понять что происходит на каждом шаге
- Используй комментарии в коде, объясняющие логику, а не очевидное

### Безопасность критична — это финтех с играми
- Сервер всегда источник истины (server-authoritative)
- Финансы через append-only ledger, никогда не UPDATE balance
- Все мутирующие операции идемпотентны
- Любая случайность в game logic — детерминированная через seed
- Никогда не доверяй данным от клиента (даты, сумма, результаты)

### Тесты обязательны для
- Game logic (≥90% покрытие правил)
- Wallet операции (включая race conditions)
- Match lifecycle (happy path + дисконнекты)

### Не делай "заранее"
- Не реализуй фичи из будущих спринтов "пока я тут"
- Не добавляй абстракции "на случай если понадобится"
- Не пиши крутые оптимизации до того, как есть проблема производительности

### Когда что-то непонятно
- Спрашивай, не угадывай
- Если запрос противоречит документам — скажи об этом до начала работы
- Если для задачи нужна информация которой нет — попроси её, не выдумывай

## Стек (зафиксирован, не предлагай менять без сильной причины)

- Backend: Node.js + TypeScript + NestJS
- Realtime: native ws + Redis pub/sub
- БД: PostgreSQL 16
- Cache/Match state: Redis 7
- Frontend: Next.js 14 + React + TypeScript + Tailwind
- Auth: own JWT (RS256) + refresh tokens
- ORM: Drizzle ORM (типизированные запросы, миграции)
- Testing: Vitest (единый для всего монорепо)
- Game libs: draughts.js (шашки), chess.js (шахматы), backgammonjs или собственная (нарды)

## Чего избегать в коде

- ❌ `any` в TypeScript — используй `unknown` или конкретные типы
- ❌ `Math.random()` в game logic — только seeded PRNG
- ❌ Прямой UPDATE на wallet баланс — только ledger
- ❌ Логика игры в `apps/api/` — она в `packages/games/{id}/`
- ❌ `body: any` в контроллерах — DTO + class-validator
- ❌ Хардкод секретов — `.env`, не в репо
- ❌ console.log в продакшен-коде — structured logger (pino/winston)
- ❌ Игнорировать ошибки `try { ... } catch {}` — логируй или брось дальше
