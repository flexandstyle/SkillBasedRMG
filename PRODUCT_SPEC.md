## 1. Что мы строим

**Skill-based RMG-платформа дуэлей**: онлайн-сервис, где два игрока играют 1-на-1 в игру, требующую навыка, на условные или реальные деньги, а платформа берёт комиссию с матча.

Не казино. Не слоты. Не букмекерка. Не "ферма". Не аркадная игра.

**Первая игра**: шашки. Архитектура с дня один рассчитана на лёгкое добавление новых игр через единый Game SDK.

## 2. Целевая аудитория

- Возраст: 18+ (строго, проверка по самоидентификации на старте, по документу при выводе в будущем).
- География на старте: РФ + СНГ. Архитектура готова к интернационализации (i18n, мультивалютность, таймзоны).
- Психотип: люди, играющие в шашки/нарды/шахматы на досуге, готовые сыграть за деньги для драйва, не профессиональные грайндеры (на старте).
- Устройства: мобильный веб (PWA) — 70%, десктоп — 30%. Дизайн mobile-first.

## 4. Принципы продукта (нерушимые)

### 4.1 Server-authoritative всегда
Клиент НИКОГДА не считает результат, валидность хода, состояние кошелька, исход матча. Клиент = рендеринг + ввод. Сервер = истина. Любой код, где клиент решает что-то про деньги или победителя, должен быть отклонён на ревью.

### 4.2 Деньги — это финтех, не игра
Любая операция с балансом — транзакция БД с явной блокировкой строки. Любое изменение баланса логируется в append-only таблицу `wallet_ledger`. Баланс никогда не считается "на лету" — это всегда сумма ledger-записей.

### 4.3 Каждый матч полностью воспроизводим
Все ходы и сидинг ГСЧ сохраняются. По replay-логу можно из стартового состояния прийти ровно к финальному. Это нужно для disputes и antifraud.

### 4.4 Платформа > одна игра
Если фича работает только для шашек — она не идёт в core. Она в `games/checkers/`. Core знает только про "матч из N ходов между двумя игроками со ставкой и победителем".

### 4.5 Антифрод не "потом"
Базовый antifraud (fingerprint, IP, паттерны побед, время на ход, **анализ совпадения ходов с движком для ВСЕХ игр, включая шашки**) — в первой же версии платных матчей. Не "v2".

### 4.6 Простота важнее красоты
PWA, а не нативные приложения. Telegram-вход, а не свой OAuth-сервер. Известные библиотеки игр, а не свои реализации правил. Каждое "сделаем сами" откладывается до момента, когда оно реально мешает.

### 4.7 Responsible gaming с дня один
Даже в виртуальной фазе: лимиты "матчей в день", отображение времени в сессии, кнопка "сделать паузу на 24 часа". При переходе на реал-мани: лимиты депозитов, self-exclusion.

## 5. Что в продукте есть

### Ядро
- Регистрация / вход (email + пароль; Telegram — после MVP).
- Виртуальный кошелёк с тремя балансами: `deposit`, `winnings`, `bonus`.
- Лобби: список открытых столов, фильтры по ставке и игре.
- Матчмейкинг: рандом или по Elo (≥100 матчей у обоих игроков). Механика постепенного расширения окна поиска Elo при долгом ожидании и push-уведомления "соперник найден". Бот-соперник как fallback при пустом лобби (только на виртуальную валюту, не на real money).
- Onboarding: обучающий матч с ботом при первом входе (для конверсии регистрация → первый матч).
- Игровая комната с реалтайм-WS, чат отключён по умолчанию (вкл. позже).
- Профиль: рейтинг, история матчей, статистика побед/поражений по играм.
- Турниры: bracket-формат, 8/16/32 игрока, гарантированный призовой фонд из entry fee.
- Реферальная программа: двухсторонний бонус виртуальной валюты.
- Лидерборды: недельные и сезонные по каждой игре.
- Replay матча: проигрывание хода за ходом с возможностью пошагать.
- Админка: ручной просмотр споров, ban пользователей, ручная корректировка балансов с логом.

### Антифрод (на этапе real money)
- Device fingerprint (FingerprintJS).
- IP + ASN-логирование, флаг при двух игроках с одного ASN.
- Лимит на матчей в день одной пары игроков (защита от перекачки денег).
- Win-rate vs new accounts detection.
- Все балансовые операции с двойной подписью при сумме > порога.

### Responsible gaming
- Self-exclusion на 24ч / 7д / 30д / навсегда.
- Лимит депозитов в день (для real money).
- Cooling-off после серии проигрышей (например, 5 подряд → опциональная пауза).

### Правила ничьей
- Ничья в шашках — нормальный исход. При ничьей обоим возвращается ставка минус половина rake (пример: оба поставили 100, призовой 200, каждый получает 90, платформа 20). Платформа всегда зарабатывает.
- Каждая игра фиксирует, возможна ли ничья в рамках Game SDK.

### Матчи с друзьями (по нику)
- На виртуальную валюту: без ограничений.
- На real money: максимум 3 матча в день с одним и тем же оппонентом (защита от перекачки денег). Сверх лимита — только рандом-матчмейкинг.

## 6. Чего в продукте НЕТ (и не будет на горизонте 12 мес)

- Слотов, рулетки, спортивных ставок, кейсов, лутбоксов.
- Игр на удачу: блэкджек, "сундучки", "верю не верю".
- Файтингов, three-in-row, аркадных игр (анти-чит дорог, skill-based трудно доказать).
- Чата с произвольным текстом без модерации (только эмодзи-набор / стикеры на старте).
- Своих NFT, своего токена, "play-to-earn" механики.
- Голосового чата.
- Стриминга матчей на старте (после 1000+ DAU).
- Покера (отдельная лицензия, отдельный анти-коллюзионный комплекс).
- Игр на >2 игрока на старте (фокусируемся на дуэлях).
- Скинов, кастомных аватарок, премиум-внешнего вида — до устойчивой выручки.

## 7. Бизнес-модель

### Источники выручки (по приоритету)
1. **Rake с матча**: 10% (минимум) — 15% (для микро-ставок). Берётся из призового фонда. Пример: оба игрока ставят 100 коинов, призовой 200, победитель получает 180, платформа 20.
2. **Entry fee турниров**: 10–15% от общего сбора.
3. **VIP-подписка** (после 500+ MAU): $10/мес, даёт пониженный rake (5%), доступ к VIP-турнирам, расширенную статистику игры.
4. **Продажа коинов** (этап виртуальной валюты): пакеты $5/$20/$50/$100.

### Чего НЕ монетизируем
- Реклама внутри продукта (отталкивает RMG-аудиторию, ломает доверие).
- "Платное удаление рекламы" (см. выше).
- Платные подсказки в игре (это P2W и убивает skill-based позиционирование).

## 9. Технологический стек (зафиксирован, не меняем без явной причины)

- **Backend**: Node.js (LTS) + TypeScript + NestJS
- **Realtime**: native ws + Redis pub/sub
- **БД**: PostgreSQL 16 (managed)
- **Cache/State**: Redis 7
- **Frontend**: Next.js 14 + React + TypeScript + Tailwind
- **Game canvas**: первая игра — react-based, для более сложных — Phaser 3 (после нард)
- **Auth**: own JWT + refresh tokens
- **Payments (real money этап)**: USDT TRC-20 через SaaS-провайдера (например, NowPayments) для MVP; собственная нода — при масштабировании
- **Monitoring**: Sentry + structured JSON logs
- **Analytics**: Posthog (self-hosted позже)
- **Deploy**: Railway/Render → AWS/GCP при росте
- **CI**: GitHub Actions
- **Game libs**: `draughts.js` для шашек, `chess.js` для шахмат

## 10. Что значит "успех MVP"

К концу 6-го месяца платформа считается успешной, если выполнено ВСЁ:
- 200+ зарегистрированных пользователей
- 30+ DAU стабильно
- D7 retention ≥ 15%
- Запущен real money pilot на USDT для ≥ 20 человек
- Ни одного известного случая успешного чита или коллюзии без обнаружения
- Среднее время "от регистрации до первого матча" < 3 минуты

Если хотя бы один пункт не выполнен — pivot, не масштабирование.
